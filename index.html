    What is the Kano Model?

    A simple way to find out which product features actually drive customer satisfaction — so you know what to build first, what to simply get right, and what to skip.

    The core idea

    Not every feature affects customers the same way. Developed by Professor Noriaki Kano in 1984, the model sorts features by how their presence or absence changes satisfaction — some delight, some are simply expected, and some make no difference at all.

    The feature types

    Must-be (M)Basics customers take for granted. If missing they're upset; if present, no extra credit. (e.g. brakes on a car)
    One-dimensional (O)More is better — satisfaction rises the more you deliver. (e.g. fuel economy)
    Attractive (A)Delighters. Their presence excites; their absence isn't missed. Great for standing out. (an unexpected extra)
    Indifferent (I)Customers don't care either way — don't spend much effort here.
    Reverse (R)Customers actually prefer this feature not to be there.
    Questionable (Q)Contradictory answers — usually a misread question. A data-quality flag, not a real category.

    How the survey works

    For each feature, respondents answer the same question two ways:

    • Functional: “How do you feel if the product has this feature?”
    • Dysfunctional: “How do you feel if it does not have it?”

    Each is answered on a 5-point scale (e.g. I like it → I expect it → I am neutral → I can tolerate it → I dislike it). Combining the two answers in a standard lookup table classifies the feature into one of the types above.

    Better & Worse scores

    For each feature the tool also computes two numbers:

    • Better (0 → 1): how much adding the feature can raise satisfaction.
    • Worse (0 → −1): how much leaving it out can drive dissatisfaction.

    Plotted together they form the classic Kano diagram, splitting features across the four quadrants.

    Why it's useful

    It turns “everyone wants everything” into a clear priority order. The classic takeaway: fulfil every must-be, stay competitive on the one-dimensionals, and stand out with a few attractives — and don't over-invest in basics or in things customers are indifferent to.

    Using this tool

    1. Build — create a survey and list the features you want to test.
    2. Collect — run the kiosk and gather responses, one person at a time.
    3. Analyze — see each feature's category, its Better/Worse scores, and the Kano diagram.

    Tip: aim for roughly 20–30+ responses before trusting the results — small samples are noisy.
